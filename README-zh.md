## 🚀 这是什么？

源自一个 Reddit 帖子，历经数千次真实会话打磨，**The Agency** 是一个精心打造的 AI Agent 人格集合。每个 Agent 都具备：

- **🎯 高度专业化**：深厚的领域专业知识（不是泛泛的提示词模板）
- **🧠 人格驱动**：独特的语气、沟通风格和工作方法
- **📋 交付物导向**：真实的代码、流程和可衡量的成果
- **✅ 生产就绪**：经过实战检验的工作流和成功标准

**可以这样理解**：就像组建你的梦之队，只不过队员是永不休息、从不抱怨、始终交付的 AI 专家。

## 📖 Agent 设计哲学

每个 Agent 的设计遵循以下原则：

1. **🎭 强烈个性**：不是泛泛的模板 — 有真实的性格和声音
2. **📋 明确交付物**：具体的产出，不是模糊的指导
3. **✅ 成功指标**：可衡量的成果和质量标准
4. **🔄 经过验证的工作流**：一步步的流程，行之有效
5. **💡 学习记忆**：模式识别与持续改进

## ⚡ 快速开始

### 方式一：在 Claude Code 中使用

#### Windows 安装（Git Bash — 推荐）

安装 [Git for Windows](https://git-scm.com/downloads/win) 后，打开 Git Bash，进入项目根目录：

```bash
# 直接安装所有 Agent — Claude Code 无需 convert 步骤
./scripts/install.sh --tool claude-code

# 只安装特定分类
./scripts/install.sh --tool claude-code --division engineering,design

# 安装单个 Agent
./scripts/install.sh --tool claude-code --agent engineering-frontend-developer
```

> **注意：** Claude Code 直接读取 `.md` 文件，**不需要** `convert.sh`。`convert.sh` 仅为其他工具（Cursor、Copilot、Gemini CLI 等）生成集成文件，详见[多工具集成](#-多工具集成)。

Agent 会被安装到 `%USERPROFILE%\.claude\agents\`（即 `C:\Users\<用户名>\.claude\agents\`）。

> **Git Bash 是什么？** Git Bash 是 Git for Windows 自带的类 Linux 终端，支持 `bash` 脚本（`.sh` 文件）。它是 Windows 上运行本项目脚本最简单的方式，无需安装 WSL 或 Cygwin。

#### Windows 安装（PowerShell / CMD — 手动复制）

如果没有 Git Bash，可以直接复制 Agent 文件。打开 **PowerShell** 或 **命令提示符**：

```powershell
# 创建目录（如果不存在）
if (-not (Test-Path $env:USERPROFILE\.claude\agents)) {
    New-Item -ItemType Directory -Path $env:USERPROFILE\.claude\agents
}

# 复制所有 Agent 文件
Copy-Item D:\PycharmProjects\agency-agents\engineering\*.md $env:USERPROFILE\.claude\agents\
Copy-Item D:\PycharmProjects\agency-agents\design\*.md $env:USERPROFILE\.claude\agents\
```

> **注意：** 手动复制方式不会更新 `AGENTS.json` 索引。推荐使用 Git Bash + 脚本方式，可以享受完整的安装/卸载/更新功能。

#### Linux / macOS

```bash
# 将所有 Agent 安装到 Claude Code 目录
./scripts/install.sh --tool claude-code

# 或者手动复制某个分类（如果你只需要一个部门）
cp engineering/*.md ~/.claude/agents/
```

#### 验证安装

```bash
# 查看已安装的 Agent
./scripts/install.sh --list-installed --tool claude-code
ls ~/.claude/agents/
```

如果能看到 `.md` 文件列表，说明安装成功。

#### 在 Claude Code 中使用

在对话中直接激活任意 Agent：

```
激活 Software Architect，帮我设计一个微服务架构
```

或使用 Claude Code 的 `/agents` 命令浏览已安装的 Agent 列表。

#### 卸载

```bash
# 卸载全部
./scripts/install.sh --uninstall --tool claude-code

# 卸载单个 Agent
./scripts/install.sh --uninstall --agent engineering-frontend-developer --tool claude-code
```

### 方式三：在其他工具中使用（GitHub Copilot、Antigravity、Gemini CLI、OpenCode、OpenClaw、Cursor、Aider、Windsurf、Kimi Code、Codex、Osaurus、Hermes）

```bash
# 第 1 步 — 为所有支持的工具生成集成文件
./scripts/convert.sh

# 第 2 步 — 交互式安装（自动检测已安装的工具）
./scripts/install.sh

# 或直接指定目标工具
./scripts/install.sh --tool antigravity
./scripts/install.sh --tool gemini-cli
./scripts/install.sh --tool opencode
./scripts/install.sh --tool copilot
./scripts/install.sh --tool openclaw
./scripts/install.sh --tool cursor
./scripts/install.sh --tool aider
./scripts/install.sh --tool windsurf
./scripts/install.sh --tool kimi
./scripts/install.sh --tool codex
./scripts/install.sh --tool osaurus
./scripts/install.sh --tool hermes
```

**只安装你需要的团队**（不是每个人都需要全部 16 个部门）：

```bash
./scripts/install.sh                                    # 交互式向导：选择工具 + 团队
./scripts/install.sh --tool claude-code --division engineering,security
./scripts/install.sh --tool cursor --agent frontend-developer,ui-designer
./scripts/install.sh --list teams                       # 查看所有团队及 Agent 数量
./scripts/install.sh --tool opencode --division engineering --dry-run
```

> **OpenCode 注意事项：** OpenCode 运行时目前只能注册约 119 个 Agent，超出的会被静默丢弃（[上游 bug](https://github.com/anomalyco/opencode/issues/27988)）。使用 `--division` 安装子集可以保持在限制以内。当选择超出限制时，安装器会给出警告。

## 🔌 多工具集成

The Agency 原生支持 Claude Code，并提供转换和安装脚本，让你可以在所有主流 AI 编程工具中使用相同的 Agent。

### 支持的工具

- **[Claude Code](https://claude.ai/code)** — 原生 `.md` Agent，无需转换 → `~/.claude/agents/`
- **[GitHub Copilot](https://github.com/copilot)** — 原生 `.md` Agent，无需转换 → `~/.github/agents/` + `~/.copilot/agents/`
- **[Antigravity](https://github.com/google-gemini/antigravity)** — 每个 Agent 生成一个 `SKILL.md` → `~/.gemini/antigravity/skills/`
- **[Gemini CLI](https://github.com/google-gemini/gemini-cli)** — `.md` Agent 文件 → `~/.gemini/agents/`
- **[OpenCode](https://opencode.ai)** — `.md` Agent 文件 → `.opencode/agents/`
- **[Cursor](https://cursor.sh)** — `.mdc` 规则文件 → `.cursor/rules/`
- **[Aider](https://aider.chat)** — 单个 `CONVENTIONS.md` → `./CONVENTIONS.md`
- **[Windsurf](https://codeium.com/windsurf)** — 单个 `.windsurfrules` → `./.windsurfrules`
- **[OpenClaw](https://github.com/openclaw/openclaw)** — 每个 Agent 生成 `SOUL.md` + `AGENTS.md` + `IDENTITY.md`
- **[Qwen Code](https://github.com/QwenLM/qwen-code)** — `.md` SubAgent 文件 → `~/.qwen/agents/`
- **[Kimi Code](https://github.com/MoonshotAI/kimi-cli)** — YAML Agent 配置 → `~/.config/kimi/agents/`
- **[Codex](https://developers.openai.com/codex/overview)** — TOML 自定义 Agent → `~/.codex/agents/`
- **Osaurus** — `SKILL.md` 技能文件 → `~/.osaurus/skills/`
- **[Hermes](integrations/hermes/README.md)** — lazy-router 插件 → `~/.hermes/plugins/`

### ⚡ 快速安装

**第 1 步 — 生成集成文件：**
```bash
./scripts/convert.sh
# 更快（并行，输出顺序可能不同）：./scripts/convert.sh --parallel
```

**第 2 步 — 安装（交互式，自动检测你的工具）：**
```bash
./scripts/install.sh
# 更快（并行，输出顺序可能不同）：./scripts/install.sh --no-interactive --parallel
```

安装器会扫描系统上已安装的工具，显示复选框界面，让你精确选择要安装的内容：

```
  +------------------------------------------------+
  |   The Agency -- 工具安装器                       |
  +------------------------------------------------+

  系统扫描: [*] = 在本机上检测到

  [x]  1)  [*]  Claude Code     (claude.ai/code)
  [x]  2)  [*]  Copilot         (~/.github + ~/.copilot)
  [x]  3)  [*]  Antigravity     (~/.gemini/antigravity)
  [ ]  4)  [ ]  Gemini CLI      (~/.gemini/agents)
  [ ]  5)  [ ]  OpenCode        (opencode.ai)
  [ ]  6)  [ ]  OpenClaw        (~/.openclaw/agency-agents)
  [x]  7)  [*]  Cursor          (.cursor/rules)
  [ ]  8)  [ ]  Aider           (CONVENTIONS.md)
  [ ]  9)  [ ]  Windsurf        (.windsurfrules)
  [ ] 10)  [ ]  Qwen Code       (~/.qwen/agents)
  [ ] 11)  [ ]  Kimi Code       (~/.config/kimi/agents)
  [ ] 12)  [ ]  Codex           (~/.codex/agents)
  [ ] 13)  [ ]  Osaurus         (~/.osaurus/skills)
  [ ] 14)  [ ]  Hermes          (~/.hermes/plugins)

  [1-14] 切换   [a] 全选   [n] 取消全选   [d] 仅检测到的
  [Enter] 安装   [q] 退出
```

**或直接安装指定工具：**
```bash
./scripts/install.sh --tool cursor
./scripts/install.sh --tool opencode
./scripts/install.sh --tool openclaw
./scripts/install.sh --tool antigravity
./scripts/install.sh --tool codex
./scripts/install.sh --tool osaurus
./scripts/install.sh --tool hermes
```

**非交互式（CI/脚本）：**
```bash
./scripts/install.sh --no-interactive --tool all
```

**并行加速** — 在多核机器上，使用 `--parallel` 让每个工具并行处理。各工具的输出顺序是不确定的。同时适用于交互式和非交互式安装：例如 `./scripts/install.sh --interactive --parallel`（先选择工具，再并行安装）或 `./scripts/install.sh --no-interactive --parallel`。并行任务数默认为 `nproc`（Linux）、`sysctl -n hw.ncpu`（macOS）或 4；可通过 `--jobs N` 覆盖。

```bash
./scripts/convert.sh --parallel                    # 并行转换所有工具
./scripts/convert.sh --parallel --jobs 8           # 限制并行任务数
./scripts/install.sh --no-interactive --parallel   # 并行安装所有检测到的工具
./scripts/install.sh --interactive --parallel      # 选择工具后并行安装
./scripts/install.sh --no-interactive --parallel --jobs 4
```
